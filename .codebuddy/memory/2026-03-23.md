### 文件变更

## 修复：Garfish 沙箱 ESM 兼容性错误

### 问题
- 生产构建后 Garfish 加载 feed 子应用报错：`Cannot use import statement outside a module`
- 根因：Garfish VM 沙箱通过 `eval()` 执行脚本，不支持 ESM 的 `import`/`export` 语法
- Vite 构建产物默认为 ESM 格式，dev 模式已通过 `isDev && sandbox.open=false` 规避，但 production 模式未处理

### 修复
- `GarfishContainer.tsx` 中统一关闭 Garfish VM 沙箱（dev + production），不再区分环境
- 样式隔离由 CSS Modules / Tailwind scoping 保障，不依赖 Garfish 沙箱
- 清理了不再使用的 `isDev` 变量

### 文件变更
- `apps/main/src/components/GarfishContainer.tsx` — 统一关闭沙箱，移除环境判断逻辑

## 修复：manualChunks 与 pnpm 不兼容导致 .pnpm chunk

### 问题
- feed 子应用构建后产生 `.pnpm-CASvzYLO.js` (553KB)，所有依赖被打到一个隐藏文件
- 根因：`manualChunks(id)` 中 `id.split('node_modules/')[1].split('/')[0]` 取到的是 `.pnpm` 而非真实包名
- pnpm 的 node_modules 路径格式为 `node_modules/.pnpm/<pkg>@<ver>/node_modules/<real-pkg>/...`，取第一个 `node_modules/` 之后的首段是 `.pnpm`

### 修复
- 改为取 `parts[parts.length - 1]`（最后一个 `node_modules/` 之后的真实包名）
- 增加 scoped 包（`@scope/pkg`）的处理逻辑
- 同时修复了 main 和 feed 两个应用的配置

### 文件变更
- `apps/feed/vite.config.ts` — 修复 manualChunks pnpm 路径解析
- `apps/main/vite.config.ts` — 修复 manualChunks pnpm 路径解析

## 优化：manualChunks 策略从按包名拆分改为按类别合并

### 问题
- 修复 pnpm 路径后，按包名一一拆 chunk 导致碎片化严重
- main: 93 个 JS chunk (1.2MB)，feed: 75 个 JS chunk (756KB)
- 大量空壳文件：`mini-decimal` 1字节、`qrcode` 30字节、`rc-tree` 197字节（只有 import 语句）
- HTTP 请求爆炸，每个碎片都有额外开销

### 优化策略
- `vendor-react`: React + ReactDOM + scheduler + react-is
- `vendor-antd`: antd + 所有 rc-* + @ant-design/* + @rc-component/*
- `vendor-garfish` (仅 main): garfish + @garfish/*
- `vendor-router` (仅 main): react-router + @remix-run/router
- `vendor-misc`: 其余第三方依赖
- 页面 chunk 不变

### 优化效果
| 指标 | main (优化前) | main (优化后) | feed (优化前) | feed (优化后) |
|------|-------------|-------------|-------------|-------------|
| chunk 数 | 93 | **10** | 75 | **5** |
| 总大小 | 1.2MB | **964KB** | 756KB | **556KB** |

### 文件变更
- `apps/feed/vite.config.ts` — manualChunks 按类别合并
- `apps/main/vite.config.ts` — manualChunks 按类别合并（额外拆 garfish/router）

## 优化：微前端共享依赖 — 子应用不再重复打包 React

### 问题
- main 和 feed 独立打包，React/ReactDOM (142KB) 被重复打包
- 用户加载主应用后进入子应用，浏览器要下载两份 React
- 多个 React 实例还可能导致 hooks 规则冲突

### 技术方案
**主应用注入全局变量 + 子应用 Rollup 虚拟模块插件**

1. 主应用 `shared-deps.ts`：将 React/ReactDOM 挂载到 `window.__SHARED_REACT__` 等全局变量
2. 子应用 `garfish-shared-deps` Vite 插件：
   - `resolveId`: 拦截 `import "react"` → 返回虚拟模块 ID
   - `load`: 为虚拟模块生成 `const __mod = window.__SHARED_REACT__; export { useState, ... }` 
3. 构建产物中 `import "react"` 完全消失，替换为 `window.__SHARED_REACT__`

### 为什么不用 Rollup external + globals？
- Vite 默认输出 ESM 格式，Rollup 的 `globals` 只在 UMD/IIFE 下生效
- ESM 下 `external` 会保留 `import "react"` 裸模块引用，浏览器无法解析

### 为什么不共享 Antd？
- Antd 是纯 UI 组件，多份实例不冲突（不像 React hooks 需要单例）
- tree-shaking 后每个子应用只打包实际用到的组件
- 全量 `import * as antd` 会导致膨胀到 2MB+（从 487KB → 2.2MB），抵消共享收益

### 优化效果
| 指标 | 优化前 (main+feed) | 优化后 (main+feed) | 节省 |
|------|-------------------|-------------------|------|
| feed 总大小 | 556KB | **404KB** | **-152KB (-27%)** |
| feed vendor-react | 142KB | **0** (共享) | **-142KB** |
| 总下载量 | 1.5MB | **1.3MB** | **-200KB** |

### 文件变更
- `apps/main/src/lib/shared-deps.ts` — 新建：将 React/ReactDOM 挂载到 window 全局变量
- `apps/main/src/main.tsx` — 在入口最顶部导入 shared-deps（确保子应用加载前就绪）
- `apps/feed/vite.config.ts` — 新增 garfish-shared-deps Rollup 插件（虚拟模块方案）
