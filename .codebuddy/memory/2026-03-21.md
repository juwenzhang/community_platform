## 类型错误修复

### vite.config.ts 多实例类型冲突
- **问题**：pnpm 因 peer deps 差异解析出 3 个 vite@6.4.1 实例（jiti@1.21.7 vs jiti@2.6.1, ±@types/node），`@vitejs/plugin-react` 返回的 Plugin 类型与项目 vite 的 Plugin 类型名义不兼容
- **根因**：`vite.config.ts` 不在 `tsconfig.json` 的 `include: ["src/**/*"]` 范围内，IDE 对其使用 inferred project，类型解析路径不可控
- **修复**：为 feed 和 main 应用创建 `tsconfig.node.json`（覆盖 `vite.config.ts`），`tsconfig.json` 通过 `references` 引用
- 新增文件：`apps/feed/tsconfig.node.json`, `apps/main/tsconfig.node.json`
- 修改文件：`apps/feed/tsconfig.json`, `apps/main/tsconfig.json`（添加 references）

### tsconfig.node.json composite 修复
- **问题**：Project References 要求被引用项目必须 `"composite": true` 且不能 `"noEmit": true`
- **修复**：为 feed 和 main 的 `tsconfig.node.json` 添加 `"composite": true`，移除 `"noEmit": true`

### tsconfig 统一继承层级重构
- **动机**：各子项目 tsconfig 大量重复配置，改为根目录维护基础配置 + 子项目 extends
- **新增根配置**：
  - `tsconfig.base.json` — 公共基础（target/module/strict/isolatedModules 等）
  - `tsconfig.app.json` — React 应用（extends base + jsx: react-jsx）
  - `tsconfig.node.json` — Vite 配置文件（extends base + composite: true）
  - `tsconfig.lib.json` — 共享库（extends base + declaration/declarationMap/sourceMap）
- **简化子项目**：
  - `apps/feed/tsconfig.json` → extends `../../tsconfig.app.json`，仅保留 outDir/rootDir
  - `apps/feed/tsconfig.node.json` → extends `../../tsconfig.node.json`，仅保留 include
  - `apps/main/tsconfig.json` → extends `../../tsconfig.app.json`，保留 outDir/rootDir/paths
  - `apps/main/tsconfig.node.json` → extends `../../tsconfig.node.json`，仅保留 include
  - `packages/shared-types/tsconfig.json` → extends `../../tsconfig.lib.json`，保留 jsx/outDir/rootDir
- **注意**：`outDir`/`rootDir` 等路径相关选项不能放在根配置里（它们相对于配置文件位置解析），必须由子项目各自指定
