## OpenSpec: connect-protocol-integration — 全部 12 项任务实现完成 ✅

### 概要
将 Gateway 从 REST proxy 升级为 gRPC-Web BFF 层，前端从裸 fetch + 手写类型改为 Connect 客户端 + proto 生成类型，实现全链路 Protobuf 类型安全通信。

### 核心技术决策
1. **tonic-web + GrpcWebLayer**：Rust 生态无 Connect Protocol 服务端实现，采用 gRPC-Web 协议作为替代
2. **Routes::into_axum_router()**：tonic 0.14 提供 `tonic::service::Routes::into_axum_router()` 将 gRPC 服务转为 axum Router，实现 gRPC-Web + REST health check 同端口共存
3. **LayerExt::named_layer()**：使用 `tonic::service::LayerExt` 将 `GrpcWebLayer` 应用到 gRPC 服务
4. **createGrpcWebTransport**：前端用 `@connectrpc/connect-web` 的 gRPC-Web Transport（而非 Connect Transport）
5. **baseUrl: '/'**：Transport 使用相对路径，通过 Vite proxy / Nginx 反代，代码环境无关
